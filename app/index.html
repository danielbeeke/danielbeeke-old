Drupal

7 jaar Drupal, begonnen met Drupal 6 met het bouwen van websites, met een bestaand thema of een subthema, naar het maken van complexe maatwerk applicaties met veel custom modules en altijd een volledig hand made theme.

Door met Drupal te werken heb ik geleerd om code te lezen bij gebrek aan (goede) documentatie. Daarmee doel ik op de vele contrib modules, die vaak iemand enthousiast in zijn of haar avond uren bij elkaar hackt.

Binnen Drupal ben ik voor meerdere langlopende producten de architect geweest. Ook heb ik in team verband classen structuren uitgedacht voor grote complexe functionaliteiten.

Binnen Drupal en in het algemeen heb ik geleerd om functionaliteiten te scheiden en nogmaals te schieden en nog weer op te delen.

Drush make en installatie profielen ken ik. Ik heb meerdere projecten als installatie profiel gemaakt.

jQuery

Met jQuery knutsel ik van alles aan elkaar. De grafiek die je op deze site ziet, is een zelfgemaakte jQuery plugin. Ik vind deze erg praktisch. Mooie kleine gecomponentiseerde pakketjes.

Binnen jQuery weet ik ook veel van AJAX, CORS en headers die nodig zijn om een losgekoppelde frontend app te laten querien op een backend API.

Ik heb meerdere apps gebouwd met vooral veel jQuery en verschillende bibliotheken.

Sass

Sass vind ik prachtig, mooier dan less. Mixins, en soms een ruby scriptje om gedaan te krijgen zoals ik het wil. Eerst leek het me omslachtig, iets je css te laten compileren maar na mijn eerste project in sass was ik verkocht. Ik heb meerdere grid systemen gebruikt en ken veel 'gem'etjes. Het laatste jaar werk ik met singularity, een grid systeem dat gewoon het beste werkt op dit moment.

Yeoman en grunt

Awesome. Even snel een klein site'tje maken, met 20 bibliotheken, een beetje sass en wat jQuery. Yeoman helpt om dat te organiseren, te compileren en uiteindelijk om te deployen. Yeoman heb ik meerdere malen gebruikt in combinatie met github pages. Geen hostingkosten en meteen ook een CDN. De site die je nu bekijkt is ook met Yeoman gemaakt, bekijk de broncode maar eens.

Grunt ben ik aardig in thuis, ik heb voor een aantal applicaties custom taken geschreven. Voor bijna ieder workflow probleem is wel een grunt module te vinden.

Door met Yeoman te werken weet ik ook hoe je de dependecy hell van node js kunt tackelen. NVM (Node version manager) en RVM (ruby version manager) gebruik ik ook.

Node JS

Javascript is langzamerhand mijn favoriete taal geworden. Ik heb eens een tooltje gemaakt die ons online klanten systeem opende met de juiste klantkaart als die klant ons belde. Ook heb ik meerdere scrapers in node geschreven. In de toekomst zou ik graag meer met node willen doen. Ik denk dat de toekomst naar talen gaat die en client en server side werken.

Een voorbeeld van een nodejs tooltje die ik gemaakt heb, er was een dataset van scholen en hun beoordelingen. Ik heb van iedere school een logo gehaald, adres gegevens en de link van hun website. Ook heb ik het adres gegeocode.

Meteor JS

The new kid on the block. Een prachtig systeem dat zich gedraagt als een soort van MVC maar dan niet voor 1 gebruiker maar voor alle gebruikers tegelijkertijd. Ik bedoel te zeggen dat states meteen doorgestuurd worden naar alle clients. Als hobby project heb ik er een appje meegemaakt. Verder ook een aantal kleine tooltjes gebouwd. Erg leuk en schoon, simpel. In productie tot nog toe erg lastig schaalbaar, sockets zijn wel zwaar voor de server en zelf hosten is ook nog niet heel makkelijk.

Ember JS

Een javascript MVC. Meerdere keren mee ge-experimenteerd, 1 app in gemaakt, helaas tot nog toe iedere keer bij het gebruiken van een javascript mvc uitgekomen bij clean javascript, het heeft volgens mij wel echt zijn use cases. Als ik een javsacript MVC zou moeten kiezen dan zou ik kiezen voor Ember ipv Angular. Vooral de specificaties en aannames die angular maakt over de backend vind ik erg mooi. Wanneer je volgens deze conventies bouwt kun je heel simpel en ordelijk alles consumen met Ember.

Angular JS

1 jaar geleden was dit project in de slop geraakt, een lange issue queue, weinig support, vastlopende site. Complexe concepten, zonder heldere uitleg. Nu zijn de issues ingehaald maar ik ben verder gaan kijken bij Ember JS en Meteor JS.

Ik heb een aantal beginners dingen gemaakt, een ToDo appje, en een rekenschematje om te budgeteren.

Inkscape

Mijn favoriete vector editor. Ik weet feiloos mijn weg in Inkscape. Sneltoesten, muiskliks, vanuit mijn grafische interesse heb ik Inkscape geleerd, ik heb veel logo's, posters, websites, folders ontworpen in Inkscape.

Ik heb twee jaar als ZZP'er gewerkt waarbij ik drukwerk en websites maakte. Ik gebruikte toen uitsluitend open-source en dus voor grafisch werk Inkscape.

Onlangs heb ik voor het eerst animated svg's gemaakt, met onder andere Inkscape.

MySQL en MariaDB

Mijn SQL ervaring is vooral Drupal based. Ik heb views omgevormd en sub queries toegevoegd. Daarvoor heb ik geleerd om SQL te schrijven. Vaak werk ik bij complexe modificaties in views met een sql query als start. Ik weet prima mijn weg te redden in sql.

GIT

Van GIT ken ik de basics en weet ik verschillende branch modellen. Ik heb GIT bij veel projecten gebruikt in een team. Branches, remotes, commits begrijp ik prima, rebasen is als zwarte magie voor me.

QGis

Eigenlijk snap ik weinig van Qgis, wel vind ik het erg vet. Ik kan data rechttrekken, en modificeren. Ik heb QGis vooral ondersteunend aan mn kaart projectjes gebruikt. Ik heb bijvoorbeeld de zee dieptes (bathymetry) lagen geconverteerd zodat ik ze in Tilemill kon gebruiken. Ook heb ik een keer een postgres server met OpenStreetMap opgezet, en data bekeken.

Leaflet

Leaflet vind ik erg vet, ik heb een aantal plugins geschreven voor Leaflet. Een publieke plugin is: Leaflet Popout Het heeft een prachtige architectuur voor een javascript bibliotheek. Ik heb het gebruikt bij al mijn kaartjes. Een tweetal voorbeelden: Inbraken in uw buurt en Drupal Shops (data heb ik gescraped).

Tilemill

Tilemill vind ik een mooie tool om mee te werken. Ik had tilemill 1 achter een publieke url draaien zodat ik de cache laag kon gebruiken voor productie kaarten. Ook heb ik dit met Tilemill 2 gedaan, dit zorgt ervoor dat ik bijvoorbeeld meerdere kaarten automatisch kan genereren op verschillende URL's. Zie deze land selector.

Technisch gezien is dit een heel zwaar proces als je de land vectoren zou laden. Daarom word de muis positie op een raster gezet, daarna wordt er een query gedaan op een postgres database die kijkt in welke geo polygon de muis staat, waarna er een land code wordt terug gestuurd. Dit zal dus ook prima werken op een mobieltje etc.

Fedora Core

De eerste serieuze linux distributie waar ik kennis mee maakte. Ik heb Fedora 3 en 4 gebruikt. Ik ben op mn 14de overgestapt op linux. Een kennis vertelde dat het leuk was en toen moest ik het uitproberen.

Ik heb vaak met mn handen in mijn haar gezeten omdat ik de x.conf weer stuk had. Linux voor de desktop is veel volwassener geworden sinds toen.

Ubuntu

Ubuntu gebruik ik nu. Ik heb het zien groeien van 1 van de gebruiksvriendelijke distro's naar een prima desktop omgeving. Ik werk graag op linux, het instaleren van een development omgeving kan ik binnen een ochtend waar dat op een mac veel langer duurt. Dan doel ik op geklooi met ruby en node js maar ook apache etc.

Aegir

Een hostingpakket voor Drupal door Drupal.

Ik gebruik een lokale Aegir installatie om alle development sites te managen. Ik heb meerdere plugins geschreven voor aegir. Poseidon was een plugin met phantomJS zodat je thumbs had in je beheer interface.

Ook heb ik veel tijd gestoken in het begrijpen hoe je een volledig geautomatiseerde OTAP kunt hebben in Aegir.

Barracuda

Barracuda is een hostingplatform met Aegir boven op een linux distro naar keuze. Ik heb een soort haat-liefde verhouding met Barracuda. Het werkt goed, is stabiel en heel erg veilig. Nadelen zijn dat voor ieder exotisch dingetje er een goede manier gezocht moet worden om het te laten werken.

Het is debian + nginx php-fpm + redis + MariaDB. Er is heel veel gemod en getuned. Ook heeft het sandboxes per linux user die dan weer verschillende sites kan hosten. Een pracht systeem met hoge complexiteit.

PostgreSQL

Voor mijn kaart projectjes gebruik ik PostgresQL en PostGIS. Ik heb de cartoDB stack weleens aan de praat gekregen lokaal. Daarin zit een component om Postgres te exposeren middels een REST API. Dus met javascript objecten ophalen binnen een bepaalde regio. Ik zou graag hier meer mee willen werken.

Ook heb ik in Postgres een keer een kopie gemaakt van de OpenStreetMap database.
